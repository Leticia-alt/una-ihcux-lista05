# una-ihcux-lista05
 As heurísticas aprendidas e aplicadas nesse sistema são a visibilidade do status, que é mostrada quando o usuário é informado dos processos que estão acontecendo dentro de seu código, por exemplo, o uso de Thread.Sleep + “...” para simular processamento; a prevenção de erros, com o uso do try/catch para capturar erros, o destaque visual da mensagem usada para informar o erro cometido; e a estética e design minimalista, com o uso das cores amarelo (título), verde (resultado) e vermelho (erro), além de separadores visuais (====, ----) e a exibição limpa do resultado com 2 casas decimais, o que facilita a leitura e torna o sistema mais intuitivo.
